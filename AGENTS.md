# AGENTS.md

## 目的
このリポジトリは、macOS / Linux 向けの GUI オーディオコンバーターを実装するためのものです。  
WAV / AIFF の書き出し、サンプルレート変換、bit depth 変換を行います。

## 技術方針
- 言語: C++20
- GUI: Slint
- 音声ファイル I/O: libsndfile
- サンプルレート変換: r8brain-free-src
- ビルド: CMake + Ninja

## 参照順序
実装前に以下を読むこと。

1. `spec.md`
2. `architecture.md`
3. `setup.md`
4. `README.md`

仕様判断は `spec.md` を最優先とする。  
技術構成や責務分割は `architecture.md` に従う。  
セットアップやローカルビルド方法は `setup.md` に従う。

## 実装の原則
- 仕様にない挙動を勝手に追加しない
- UI は 1 画面完結のシンプル構成を維持する
- v1 ではモノラル / ステレオに対応する
- 多チャンネル、非対応形式はスキップしてログに記録する
- 同一条件のファイルはスキップしてログに記録する
- 上書き保存は必ず一時ファイル経由で行う
- 失敗時に元ファイルを壊さない
- 1 ファイルの失敗で全体停止しない
- Python / Node.js を前提にしない
- Slint, libsndfile, r8brain-free-src の責務を混ぜない

## 役割分担
- `ui/`
  - Slint UI
  - 画面イベント
- `src/core/`
  - 設定検証
  - 変換ジョブ管理
  - ログ管理
- `src/audio/`
  - libsndfile と r8brain-free-src の橋渡し
- `src/util/`
  - パス処理
  - prefix / postfix 処理
  - 一時ファイル処理

## 変更時のルール
- 大きな仕様変更を行う前に、まず `spec.md` を更新する
- モジュール責務が変わる場合は `architecture.md` を更新する
- ローカル環境依存の内容は `setup.md` に書く
- 個人環境専用の設定はリポジトリに含めない
- `CMakeUserPresets.json` はローカル専用として扱う

## コーディング方針
- 小さく分割する
- 1 ファイルに責務を詰め込みすぎない
- GUI 層に変換ロジックを直接書かない
- 変換ロジックは UI 非依存に保つ
- ログ文言は一貫させる
- 例外系を先に処理する
- パス操作とファイル操作は安全性を優先する
- 将来の機能追加を見越しても、v1 時点で不要な抽象化は増やしすぎない

## ビルドと確認
macOS のローカル例:

    cmake --preset local-llvm
    cmake --build --preset local-llvm

Linux の基本例:

    cmake -S . -B build -G Ninja
    cmake --build build

変更後は最低限以下を確認すること。
- configure が通る
- build が通る
- 既存仕様を壊していない
- UI から変換実行までの流れが破綻していない

## 禁止事項
- 仕様未確定の機能を勝手に実装する
- UI の見た目を勝手に大きく変える
- ログ仕様を黙って変える
- スキップ条件や安全仕様を黙って変える
- 入出力形式を勝手に追加・削除する
- 外部依存を無断で増やす
- 一時ファイルを使わずに上書き保存する

## 優先順位
判断に迷ったら以下を優先する。

1. 元ファイルを壊さない
2. `spec.md` に一致させる
3. シンプルさを保つ
4. 拡張しやすい分離を保つ
5. 実装量を増やしすぎない
