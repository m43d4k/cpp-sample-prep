# SamplePrep 要件定義 v1

## 1. 概要
macOS 向けの GUI オーディオコンバーターを作成する。
Linux 対応はひとまず見送る。
起動すると、シンプルなチェック項目と出力先を設定できる画面を表示し、指定条件で音声ファイルを一括変換できる。

---

## 2. 対応OS
- macOS

---

## 3. 画面要件
起動時に 1 画面で完結するシンプルな UI を表示する。

### 画面上の主な項目
- 入力ファイル群または入力フォルダの選択
- 出力方式の選択
- 出力先の指定
- ファイル名ルールの指定
- サンプリングレートの選択
- 出力フォーマットの選択
- bit depth の選択
- 使用 CPU コア数の選択
- 実行ボタン
- ログ表示
- 進捗表示

---

## 4. 入力仕様

### 4.1 読み込み
- v1 の入力形式は WAV / AIFF / FLAC / MP3 / OGG / CAF とする
- 入力は複数ファイルまたは単一フォルダを受け付ける
- 入力ファイル群 / 入力フォルダはネイティブダイアログで指定できる
- macOS では入力ファイル群 / 入力フォルダをドラッグアンドドロップでも指定できる
- フォルダ入力時は指定フォルダ配下を再帰的に対象とする
- ただし v1 の変換対象はモノラルまたはステレオファイルとする

### 4.2 v1 の制約
- 入力方法は以下のいずれか
  - 複数ファイル
  - 単一フォルダ
- モノラル / ステレオ対応
- 多チャンネルは対象外
- フォルダ入力は再帰

### 4.3 非対応入力の扱い
- 非対応形式はスキップし、ログに記録する
- 多チャンネル入力はスキップし、ログに記録する
- 壊れたファイルや読み込み不能ファイルはスキップし、ログに記録する

---

## 5. 出力仕様

### 5.1 出力フォーマット
- wav
- aiff

### 5.2 サンプリングレート
- 44100
- 48000
- 88200
- 96000

### 5.3 bit depth
- 8 bit PCM
- 16 bit PCM
- 24 bit PCM
- 32 bit PCM

### 5.4 使用 CPU コア数
- All
- 1
- 2
- 3
- 4
- 6
- 8
- 10

#### 挙動
- デフォルトは All とする
- All の場合は `std::thread::hardware_concurrency()` を取得する
- 実際の worker 数は `min(取得したコア数, 入力ファイル数)` とする
- `std::thread::hardware_concurrency()` が 0 を返した場合は 1 とみなす
- 並列化はファイル単位で行う
- 各ジョブは独立した input handle / output handle / resampler を持つ
- 1 ファイル内のチャンネル分割並列化は行わない

---

## 6. 出力方式

### 6.1 Overwrite original files
元ファイルを置き換える。

#### 挙動
- 一時ファイル経由で安全に置換する
- 変換成功後に元ファイルを置き換える
- 失敗時は元ファイルを保持する
- バックアップファイルは作成しない
- 出力フォーマット変更で拡張子が変わる場合は、同一ディレクトリ内で拡張子だけを出力形式に合わせて置き換える
- この場合も、一時ファイルから最終ファイルを確定した後に元ファイルを削除する

### 6.2 Use source file directory
元ファイルは残し、元ファイルと同じディレクトリに別名で出力する。

#### 必須項目
- ファイル名の変更方法として prefix または postfix を指定する

#### 挙動
- 出力先ディレクトリは各入力ファイルの親ディレクトリとする
- prefix / postfix を付けた別名で保存する
- 出力先指定フォームは使用しない

### 6.3 選択なし
元ファイルは残し、別名で出力する。

#### 必須項目
- 出力先フォルダを指定する
- ファイル名の変更方法として prefix または postfix を指定する

#### 命名ルール
- prefix / postfix はどちらか一方のみ選択可能
- 両方同時指定は不可
- 入力が単一フォルダの場合、出力先では選択フォルダからの相対フォルダ構造を維持する
- 入力が複数ファイルの場合、出力ファイルは出力先フォルダ直下に作成する

---

## 7. 同一条件の扱い

### 7.1 同一条件の定義
同一条件とは、入力ファイルの以下が指定された出力条件とすべて一致する状態を指す。

- 出力フォーマット
- サンプリングレート
- bit depth

### 7.2 挙動
- 同一条件ならスキップする
- スキップしたことをログに記録する

### 7.3 例
入力:
- WAV
- 48000 Hz
- 24 bit PCM

出力指定:
- WAV
- 48000 Hz
- 24 bit PCM

この場合は同一条件なので、変換せずスキップしてログに記録する。

---

## 8. コンテナ変更時の扱い

### 8.1 例
入力:
- WAV
- 48000 Hz
- 24 bit PCM

出力指定:
- AIFF
- 48000 Hz
- 24 bit PCM

### 8.2 挙動
この場合は同一条件ではないため、変換対象とする。

ただし、行う処理は以下の通り。
- サンプリングレート変換は行わない
- bit depth 変換は行わない
- 音声データ条件は維持する
- 出力フォーマットに合わせてコンテナを書き換える

### 8.3 解釈
- wav → aiff
- aiff → wav

のようなケースでは、基本的に「音声条件は同じまま、コンテナだけ変更する」扱いとする。

---

## 9. 変換エンジン

### 9.1 サンプルレート変換
- r8brain-free-src を使用する

### 9.2 役割分担
- r8brain-free-src はサンプルレート変換専用として使用する
- WAV / AIFF の読み書き、および PCM bit depth の処理は別レイヤーで扱う

---

## 10. 変換処理の基本フロー
1. 入力ファイルを開く
2. 入力形式が対応対象か確認する
3. モノラルまたはステレオか確認する
4. 出力条件と比較する
5. 同一条件ならスキップしてログ記録する
6. 同一条件でなければ変換を行う
7. 出力方式に応じて保存または上書きする
8. 成功 / 失敗 / スキップをログに記録する

---

## 11. ログ仕様
各ファイルについて以下のいずれかをログに記録する。

- Success
- Failed
- Skipped

### Skipped の主な理由
- 非対応形式
- 多チャンネル
- 同一条件
- 読み込み不能
- 出力不能

---

## 12. 安全仕様
- 上書き保存は一時ファイル経由で行う
- 失敗時は元ファイルを保持する
- バックアップファイルは作らない
- 1 ファイルの失敗で全体停止しない
- 処理結果はファイルごとにログ表示する

---

- 起動時にシンプルな GUI を表示
- 入力は複数ファイルまたは単一フォルダ
- フォルダ入力は再帰
- 読み込み形式は WAV / AIFF / FLAC / MP3 / OGG / CAF

- 出力は wav / aiff
- bit depth は 8 / 16 / 24 / 32 bit PCM
- サンプリングレートは 44.1 / 48 / 88.2 / 96 kHz
- 出力方式は Overwrite original files / Use source file directory / 別名保存
- 別名保存時は prefix / postfix のどちらか一方のみ指定
- 入力ファイル、入力フォルダ、出力フォルダはネイティブダイアログから選択可能
- 同一条件ならスキップしログに記録
- フォーマットだけ異なる場合は変換対象
- サンプルレート変換には r8brain-free-src を使う
- 上書きは一時ファイルのみで安全に行う
